# Proposal

Status: Deferred by the Orchestrator after Task 0007.1 verification.

Clarify how `dependsOn` should work when a production component subscribes to
more environments than its test component, or add environment-scoped dependency
support to Orun.

# Found By

Task 0007.1 Verifier.

# Related Task

Task 0007.1 / PR #34.

# Current Spec Text / Contract

`specs/repo.md` says deployable, package, and infra components must declare
`dependsOn` edges to the test components that gate them.

`specs/orun-golden-path.md` says `dependsOn` is the explicit component ordering
edge and should be used whenever ordering or required context matters.

# Repo Reality / New Information

`packages/db` subscribes to `dev`, `stage`, and `prod`. `tests/db` subscribes
only to `dev`, matching the existing `packages/contracts` / `tests/contracts`
pattern.

A direct `dependsOn` edge from `db` to `db-tests` would be ambiguous or unsound
for `stage` and `prod`, where there is no `db-tests` job instance. Orun does
not currently expose an environment-scoped dependency edge that says
`db.dev.verify` depends on `db-tests.dev.verify` while `db.stage.verify` and
`db.prod.verify` do not.

# Proposed Spec Change

Either:

1. Add environment-scoped `dependsOn` support to Orun/component manifests.
2. Clarify that test dependencies may be documented rather than declared when
   the producer and test components intentionally subscribe to different
   environment sets.

# Why This Is Needed

The current spec is stricter than the composition model can safely express for
dev-only test gates on components that also run in `stage` and `prod`.

# Impacted Files / Tasks

- `specs/repo.md`
- `specs/orun-golden-path.md`
- `stack-tectonic/**` if environment-scoped dependency support is implemented
- Current and future package/test component pairs such as `db`/`db-tests` and
  `contracts`/`contracts-tests`

# Compatibility / Migration Notes

This does not block Task 0008. Existing PR and main CI runs execute the test
components independently. A later task can either add first-class environment
scoping or clarify the documented exception without changing runtime behavior.

# Recommendation

Defer until it blocks a release-critical DAG or until a broader Orun component
contract cleanup task is scheduled.
