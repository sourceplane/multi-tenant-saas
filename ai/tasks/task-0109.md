## Architect Brief

- **Product bar:** Stripe + Linear-quality dashboard. The reveal-once
  modal is the buyer-credible moment — it must look and feel as deliberate
  as the `api-keys` reveal flow (single dialog, monospace secret block,
  copy button with success affordance, clear "this is the only time"
  language, explicit Close requires confirmation that it was copied).
- **User moment this PR improves:** an org admin lands on
  `/orgs/{orgSlug}/webhooks`, sees their webhook endpoints, opens one,
  clicks "Rotate signing secret", confirms in a destructive-styled
  dialog, and is shown the new `whsec_…` plaintext exactly once with a
  copy-to-clipboard button and a clear note about the grace window
  (timestamp + duration). Closing the modal scrubs the secret from React
  state. Reloading the page never re-shows it.
- **Free to decide (record one-line rationale in the implementer report):**
  - exact route shape under `/orgs/[orgSlug]/webhooks/**` (suggest a list
    page at `/webhooks` and a detail page at `/webhooks/[endpointId]`,
    but a single combined page with an inline drawer is acceptable if
    cleaner);
  - whether the rotate action lives on the list row, on the detail page,
    or both (suggest detail page only — keeps the destructive action
    behind one extra click, mirrors Stripe);
  - exact copy strings, microcopy for the grace-window banner, and the
    grace-window date formatter (suggest `Intl.DateTimeFormat` long form
    + relative duration like "in 24 hours");
  - whether to surface `secretVersion`, `lastUsedAt`, `disabledAt`,

## Objective

Ship the reveal-once webhook signing-secret rotation UX in the
`web-console-next` app, end-to-end, on top of the locked Task 0108
contract. One PR delivers: a Webhooks list page under the org scope,
a webhook endpoint detail page, a destructive-confirm rotate flow, and
a Stripe/Linear-quality reveal-once modal that surfaces
`{ secret, previousSecretExpiresAt, gracePeriodSeconds }` exactly once.

"Webhooks", placed adjacent to "API keys" (suggested order: after API
keys, before Config). Icon choice is implementer-free.

## Required Outcomes

1. New routes under `apps/web-console-next/src/app/(app)/orgs/[orgSlug]/webhooks/**`
   that list webhook endpoints (via `client.webhooks.listEndpoints`) and
   render endpoint detail (via the existing list response — no extra
   `getEndpoint` call required unless cleaner).
2. A "Rotate signing secret" action on the detail surface that:
   - opens a destructive-styled confirm dialog with explicit copy
     describing the dual-signature grace window;
   - on confirm, calls `client.webhooks.rotateSecret(orgId, endpointId)`
     through `wrap()` and surfaces errors via the existing toast + the
     console envelope/precondition pattern;
   - on success, opens a reveal-once modal that displays the plaintext
     `secret` in a monospace block with a single copy-to-clipboard
     button, alongside the formatted `previousSecretExpiresAt` and a
     human-readable rendering of `gracePeriodSeconds`;
   - scrubs the secret from React state when the modal closes
     (controlled state + explicit reset on close);
   - never persists the secret to any browser storage, query cache, or
     telemetry.
3. The secret value never appears outside the reveal-once modal: not in
   list rows, not in the detail surface after the modal closes, not in
   toasts, not in error envelopes, not in console-side debug logs.
4. Sidebar gains exactly one new "Webhooks" entry under the org scope.
5. The detail surface renders read-only metadata available on
   `PublicWebhookEndpoint` (at minimum: `id`, `url`, `eventTypes`,
   `secretVersion`, `createdAt`; surface `disabledAt` / `lastUsedAt`
   when present). Layout is implementer-free.
6. Empty / loading / error states are wired (use `EmptyState`, `Skeleton`,
   and the `wrap()` envelope; no raw `precondition_failed` strings reach
   the user).
7. Targeted test(s) cover at minimum the reveal-once invariant (modal
   shows secret once, secret is cleared on close) and the rotate-confirm
   gating (no rotate call without explicit confirm). Test layout is
   implementer-free; reuse the closest existing convention.
8. The endpoint when `secret` is omitted from the response (legacy
   no-encryption-key case per contract docstring) is handled
   gracefully — show a plain "rotation completed; secret was not
   returned by the server" affordance with a doc-shaped explanation,
   not a crash.

  `src/lib/session.tsx`; mirror how `api-keys/page.tsx` and

- `client.webhooks.listEndpoints(orgId)` returns
  `ListWebhookEndpointsResponse`. `client.webhooks.rotateSecret(orgId,
  endpointId)` returns `RotateWebhookSecretResponse`.

  source of truth for both the list page and the detail page (avoid an
  extra round-trip unless your chosen route shape requires one).

  render "no grace window applied" copy and explain that the previous

  timestamp + a duration ("in ~24 hours") so the operator can plan
  receiver rollout.
- For the legacy no-secret case (`secret` undefined on the response),
  see Required Outcome 8.
