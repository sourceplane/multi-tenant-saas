## Architect Brief

- Product bar: a clean `pnpm -r typecheck` baseline across the workspace.
  Today, 36 workspace projects but two test workspaces fail typecheck
  on a no-op clean checkout. That noise hides real regressions.
- The implementer has full latitude on the precise tsconfig / devDeps fix
  shape per workspace, as long as:
  - The fix is minimal — the goal is not to refactor either test
    workspace, only to clean up the typecheck baseline.
  - No production source under `apps/**` or `packages/**` is touched.
  - No new runtime dependency is added; the fix should be limited to
    `tsconfig.json`, `package.json` `devDependencies`, and (if absolutely
    necessary) one `globals.d.ts`-style ambient declaration file.
  - The actual jest test runs continue to pass — `pnpm -F
    @saas/identity-worker-tests test` and
    `pnpm -F @saas/policy-engine-tests test` both stay green.
- Recommended first attempt:
    1. For `policy-engine-tests`: ensure `@types/node` is in the
       devDependencies of that workspace's `package.json` (or root, but
       prefer per-workspace to avoid widening), or remove the explicit
       `"types": ["node"]` from its `tsconfig.json` if Node typings are
       not actually consumed by any test. Pick whichever produces the
       smaller diff.
    2. For `identity-worker-tests`: the failing files reference
       `globalThis.crypto`. The Workers runtime has it natively; under
       jest+`@types/node` the `Crypto` global may not be picked up.
       Either (a) add `"lib": ["...", "DOM"]` to the test workspace
       tsconfig (DOM lib provides `crypto`), or (b) use
       `globalThis.crypto as Crypto` cast pattern, or (c) add a one-line
       `node:crypto` import where applicable. Pick the smallest fix.
- Failure modes that invalidate the PR even if typecheck passes:
    - Adding new runtime dependencies (only devDependencies allowed).
    - Touching production source in `apps/**`, `packages/**`, `infra/**`,
      `wrangler.jsonc`, or any `component.yaml`.
    - Regressing any existing jest test pass count.
    - Modifying anything in the deferred boundaries.
