# Required Outcomes

- Wire `multi-tenant-saas` CI and/or the Terraform Orun composition to assume
  the environment-appropriate `aws-admin` role before Terraform needs AWS
  access.
- Keep credential setup visible in the Orun plan. Prefer a Terraform
  composition step using `aws-actions/configure-aws-credentials@v4` or an
  equivalent Orun-planned capability over hidden workflow shell.
- Migrate Terraform backend configuration to AWS S3:
  - bucket: `sourceplane-<env>`;
  - key: `multi-tenant-saas/<component>/terraform.tfstate`;
  - `workspace_key_prefix = "env"`;
  - `encrypt = true`;
  - `use_lockfile = true`;
  - region from `AWS_REGION` or Terraform parameters, default `us-east-1`.
- Remove or rename R2-specific backend components and docs so there is no
  active path implying Cloudflare R2 Terraform state.
- Update Terraform component manifests and READMEs to document the S3 backend,
  AWS role dependency, and secret namespace.
- Verify access to AWS through the assumed role:
  - `aws sts get-caller-identity`;
  - read/list the expected S3 state bucket for at least `dev`;
  - read and write a scoped Secrets Manager smoke secret under
    `sourceplane/multi-tenant-saas/access-smoke/dev`, then clean it up or record
    why it intentionally remains.
- Update `ai/context/*` and write the implementer report.
