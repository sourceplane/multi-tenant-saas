## Why this task exists

PR #150 (Task 0098) shipped the `packages/sdk/` workspace with full source,
tests, README, eslint config, and tsconfigs — but **no `component.yaml`**.
Every other workspace package (`contracts`, `shared`, `policy-engine`,
`notifications-client`, `db`, `testing`) carries a `component.yaml`
declaring it as an Orun `turbo-package` component with environment
subscriptions and verify/build commands. Without it:

- Orun composition (`stack-tectonic`) cannot discover the SDK package.
- CI's component-scoped change detection (`automation.triggerBindings.*`,
  `discovery.roots: packages/`) will not include `packages/sdk/**` in
  changed-component plans.
- The SDK won't appear in any Orun job graph, can't be wired into
  multi-environment verify/build flows, and won't get its own scoped
  CI lane.

This task aligns `packages/sdk` with the rest of the monorepo's component
contract — bootstrap polish, not new feature work.

## Required outcomes

1. **New file:** `packages/sdk/component.yaml`. Mirror the
   `packages/contracts/component.yaml` and `packages/shared/component.yaml`
   shape. Use:

   - `apiVersion: sourceplane.io/v1`
   - `kind: Component`
   - `metadata.name: sdk`
   - `metadata.description:` one-line description matching the SDK README's
     opening line ("Runtime-agnostic TypeScript client for the Sourceplane
     multi-tenant SaaS API." — verify the exact wording in
     `packages/sdk/README.md` and copy it).
   - `spec.type: turbo-package`
   - `spec.domain: starter-sdk` (new domain — packages currently use
     `starter-contracts`, `starter-shared`, `starter-policy`, etc.; this
     follows the `starter-<surface>` convention).
   - `spec.subscribe.environments`: `dev`, `stage`, `prod`, all with
     `profile: quick-check` (matches `contracts` and `shared`).
   - `spec.parameters.nodeVersion: "20"` (matches `contracts`/`shared`;
     do not bump to 22 — the SDK package.json doesn't pin engines, and
     the rest of the shared layer is on 20).
   - `spec.parameters.pnpmVersion: "10.12.1"`.
   - `spec.parameters.buildCommand: pnpm exec turbo run build --filter=./`
   - `spec.parameters.typecheckCommand: pnpm exec turbo run typecheck --filter=./`
   - `spec.parameters.preBuildCommand: pnpm exec turbo run test --filter=./`
     (the SDK has a real vitest suite in `src/__tests__/sdk.test.ts`;
     wire it in like `tests/contracts` does — verify suite continues to
     run by `--filter` resolving correctly).
   - `spec.labels`: `team: platform`, `layer: shared`, `surface: sdk`.
