# Required Outcomes

1. Add environment public ID handling.
   - Add `env_` public ID helpers in projects-worker, matching the existing
     `org_` and `prj_` UUID-hex pattern.
   - Parse malformed environment IDs as 404 at the route boundary.
   - Public responses, event payloads, audit descriptions, and tests must use
     public `org_`, `prj_`, and `env_` IDs where IDs are user-facing.
   - Raw UUIDs remain internal for database, policy, and event/audit scope
     columns.

2. Add environment create.
   - Route:
     `POST /v1/organizations/{orgId}/projects/{projectId}/environments`.
   - Require actor headers exactly like existing project routes.
   - Validate JSON body using the existing project create shape as the local
     pattern: `name` is required; `slug` is optional; slug rules should match
     project slug rules unless a stricter local reason is documented.
   - Parse `org_` and `prj_` IDs and return 404 for malformed IDs.
   - Confirm the parent project exists in the organization and is active before
     creating an environment. Missing or archived parent project must return
     404.
   - Use membership-worker authorization-context with actor subject and raw org
     UUID.
   - Add and use policy action `environment.create`.
   - Authorize with explicit project scope:
     `{ kind: "environment", orgId, projectId }`.
   - `environment.create` must be project-scoped in the policy engine.
   - Role semantics:
     - organization `owner`, `admin`, and `builder` may create environments;
     - organization `viewer` and `billing_admin` may not;
     - matching `project_admin` and `project_builder` may create environments;
     - `project_viewer` may not;
     - cross-org or wrong-project role facts must deny.
   - Use `createEnvironment` from `@saas/db/projects`.
   - Map slug conflicts to 409.
   - Map repository/internal failures to safe 503 responses.
   - Return `201` with `{ environment: PublicEnvironment }`.

3. Write `environment.created` event/audit atomically.
   - Environment insert and `environment.created` event/audit append must share
     the transaction-capable SQL executor pattern used by project create/archive.
   - If event/audit append fails, environment creation must not successfully
     commit.
   - Use event type `environment.created`, source `projects-worker`, category
     `projects`, and a concise audit description.
   - Event/audit scope columns should include raw `orgId`, `projectId`, and
     `environmentId`.
   - Public-facing payloads and audit descriptions should include public IDs
     and must not expose raw UUIDs, database connection details, bearer tokens,
     API keys, SQL, stack traces, or other secret material.

4. Add environment list.
   - Route:
     `GET /v1/organizations/{orgId}/projects/{projectId}/environments`.
   - Require actor headers.
   - Parse `org_` and `prj_` IDs and return 404 for malformed IDs.
   - Confirm the parent project exists in the organization and is active before
     listing environments. Missing or archived parent project must return 404.
   - Use membership-worker authorization-context before policy-worker.
   - Authorize with existing project-scoped `environment.read` and explicit
     project resource `{ kind: "environment", orgId, projectId }`.
   - Do not add `environment.list` unless verification finds a concrete
     contract blocker; the route already has explicit project scope.
   - Use cursor pagination with the established `limit`/`cursor` contract:
     default 50, max 100, opaque versioned cursor, `meta.cursor`.
   - Use `listEnvironmentsPaged(orgId, projectId, pageParams)`.
   - Return `{ environments: PublicEnvironment[] }` with public IDs.
   - Existing repository filtering for active environments should remain in
     place; do not change archive semantics.

5. Add environment get.
   - Route:
     `GET /v1/organizations/{orgId}/projects/{projectId}/environments/{environmentId}`.
   - Require actor headers.
   - Parse `org_`, `prj_`, and `env_` IDs and return 404 for malformed IDs.
   - Confirm the parent project exists in the organization and is active before
     fetching the environment. Missing or archived parent project must return
     404.
   - Use membership-worker authorization-context before policy-worker.
   - Authorize with existing `environment.read` and explicit resource:
     `{ kind: "environment", id: environmentId, orgId, projectId, environmentId }`.
   - Use `getEnvironmentById(orgId, projectId, environmentId)`, never
     environment ID alone.
   - Return `{ environment: PublicEnvironment }` with public IDs.
   - Policy denial and repository not-found must return 404.

6. Update api-edge project facade.
   - Forward the three environment routes to `PROJECTS_WORKER` after identity
     resolution.
   - Preserve query params on the list route.
   - Forward actor headers, request ID, traceparent, and idempotency-key as
     appropriate.
   - Continue not forwarding bearer tokens.
   - Forward POST body only for create.
   - Preserve existing project POST/GET/list/DELETE behavior.
   - Do not expose environment update/archive/delete or unrelated project
