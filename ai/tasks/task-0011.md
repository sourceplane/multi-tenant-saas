# Objective

Add the first Worker-safe Hyperdrive/Postgres runtime adapter seam and a
read-only operational smoke path in `api-edge` that proves the deployed Worker
can reach Supabase Postgres through the `SOURCEPLANE_DB` binding.

This task must prove runtime connectivity without adding tenant-domain business
logic, domain tables, identity flows, organization/project APIs, or writes.

# PR Boundary

# Required Outcomes

- Add a Worker-safe runtime adapter under `packages/db` that:
  - accepts a Cloudflare Hyperdrive binding;
  - uses the binding's runtime connection string, not a committed or
    environment-sourced raw database connection string;
  - uses a Worker-compatible Postgres client recommended by current Cloudflare
    Hyperdrive docs, such as Postgres.js;
  - exposes a narrow interface suitable for future repository adapters;
  - supports a read-only health/smoke query such as `SELECT 1`;
  - closes or releases client resources when the request is done;
  - redacts or normalizes errors so passwords, connection strings, tokens, and
    SQL internals are never returned to clients or written to reports;
  - remains isolated from Node-only migration runner code.
- Add focused tests for the adapter that cover:
  - missing Hyperdrive binding;
  - successful read-only query through an injected/fake SQL client;
  - query failure mapped to a safe operational status/error;
  - cleanup/disposal after success and failure;
  - no import path from the Worker adapter to runner-only modules.
- Keep `@saas/db` default exports Worker-safe. If you add a subpath export,
  verify default consumers do not pull in `pg`, AWS SDK, `node:fs`,
  `node:crypto`, or migration runner code.
- Add a read-only `api-edge` operational route or health extension that:
  - is safe when `SOURCEPLANE_DB` is absent locally;
  - uses the new adapter when the binding exists;
  - does not write to the database;
  - does not expose raw SQL errors or secret material;
  - does not introduce public tenant-domain behavior;
  - returns a stable JSON shape based on the existing health contract or a
    clearly documented operational health shape.
- The route may query only non-domain control state or a literal read-only
  expression. Acceptable examples: `SELECT 1 AS ok` or read-only migration
  control metadata. Do not query or create identity, org, project, billing,
  audit, or other domain tables in this task.
- Update app/package scripts if needed so local and Orun checks exercise the
  new adapter/route tests.
- Open a GitHub PR for the task branch.
- Write `ai/reports/task-0011-implementer.md` with final checks and the real PR
  number/URL.
