## Objective

Scaffold `packages/cli` as a first-class Orun component and ship the
B4 second-half foundation per `specs/components/13-cli-and-sdk.md`:
the CLI binary, command framework, auth flow with token storage on
keychain (with `~/.config/sourceplane/` fallback), org-context
persistence, machine-readable JSON output mode, and a small pilot of
**read-only** commands wired through `@saas/sdk` end-to-end. The
remaining write commands (`org invite`, `project create`, `env create`,
`api-key create`, `webhook create`, `usage summary`, `billing summary`,
`audit list`) become Task 0101 (CLI command fan-out) — same cadence
as Task 0098 → Task 0099.

     `--api-url` defaulting to a constant, then surfaces a one-time
     code + URL the user opens; if the spec/contracts do not yet have
     a device-flow endpoint, fall back to **token paste** flow:
     `sourceplane login` prompts for a Bearer token, validates by
     calling `client.organizations.list()`, stores on success).
     Decide and document. `whoami` reads token + cached org context,
     calls SDK `organizations.list()` to validate and surfaces
     identity + active org.

     implementations: `KeychainTokenStore` (uses `keytar` lazily,
     service `sourceplane-cli`) and `FileTokenStore`
     (`~/.config/sourceplane/credentials.json`, mode 0600 on POSIX,

     to stdout; on error emits `{ error: { code, message, requestId? } }`

     commands and a `key: value` block for single-record reads.
   - `errors.ts` — translates `SourceplaneError` subclasses (from

## Constraints

1. **Public API only.** CLI imports `@saas/sdk` (and optionally types
   from `@saas/contracts` re-exported through the SDK). It must not
   import from `apps/**`, `packages/db/**`, or any worker package.
2. **Hazard ban.** Same as Tasks 0096b–f, 0097, 0098, 0099: zero new
   `eslint-disable*`, `@ts-ignore`, `@ts-expect-error`, `as unknown as`,
   `as any` anywhere under `packages/cli/**`.
3. **Caller-owned idempotency.** When Task 0101 adds POSTs, the CLI
   must let users pass `--idempotency-key` and forward it through the
   SDK. This PR does not introduce write commands but must NOT add a
   transparent idempotency-generation layer that Task 0101 would
   inherit by accident. Stripe parity is preserved by SDK; CLI must
   not regress it.
4. **No new runtime deps in workspace root.** Any CLI runtime deps
   (e.g., `commander`, `keytar`) live under `packages/cli/package.json`
   only. `keytar` must be in `optionalDependencies` (or
   `peerDependenciesMeta.optional: true`) so install does not fail on
   environments without native build tools.
5. **Token-store file safety.** On POSIX, credentials file MUST be
   created with mode 0600 (owner read/write only) and parent directory
   with mode 0700. Verified by an integration-style test.
6. **Output stability.** JSON output mode must be deterministic given
   deterministic SDK responses. No timestamps in JSON envelopes the
   CLI itself adds. Tests must assert exact JSON shape.
7. **Same `quick-check` profile shape as `packages/sdk`.** No new
   apply lanes, no Cloudflare deploy, no Worker bindings.
8. **Branch & PR.** Work on `impl/task-0100-packages-cli-scaffold`.
   Commit, push, open PR via `gh pr create`. **`PR Number: TBD` is
   not an acceptable completed report.** Update report and any
   placeholder context with the real PR number after creation, then
   commit + push.

## Integration Notes

- `packages/sdk` already exports `Sourceplane`, the per-resource
  clients, the `Transport` class, and the typed error hierarchy. CLI
  consumes `Sourceplane` for command handlers and the typed error
  classes for error formatting. Do not bypass to `Transport` directly
  except in tests.
- `~/.config/sourceplane/` directory layout (suggested):
  - `credentials.json` (0600) — `{ apiUrl, token }` (file fallback only;
    keychain stores nothing on disk).
  - `config.json` (0644) — `{ activeOrgId?, lastApiUrl? }`.
  Both paths overrideable via `SOURCEPLANE_CONFIG_DIR` for tests.
- `packages/cli/component.yaml` is the part Orun discovery sees; the
  `quick-check` profile means PR-CI runs lint+typecheck+test on dev,
  stage, and prod environment slots without doing any deploy. Mirror
  SDK exactly.
