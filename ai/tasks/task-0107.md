## Surface

```
sourceplane webhook sign
  --secret=SECRET                 (required)
  --timestamp=UNIX_SECONDS        (required, integer string)
  [--body=PATH]                   file bytes — mutually exclusive with piped STDIN
  [--output=human|json]           default human
```

### Output shape

- `--output=human` (default):
  ```
  signature: sha256=<hex>
  timestamp: <ts>
  ```
  exit 0.
- `--output=json`:
  ```
  {"signature":"sha256=<hex>","timestamp":"<ts>"}
  ```
  exit 0.
- `UsageError` (missing flag, both `--body=PATH` + STDIN, unreadable
  file, malformed `--timestamp`) → exit 2 via `formatCliError`.

The shape MUST round-trip cleanly with `webhook verify` (the
acceptance test below covers this).

## Hard rules

1. **No network, no auth, no SDK, no `Sourceplane`, no `client.*`,
   no `fetch(`, no `/v1/`.** This command never touches the API.
2. **No `node:crypto`, no `node:buffer`, no Node-only crypto.** Sign
   ONLY through `@saas/webhook-verifier`'s `signWebhookPayload`.
3. **Body bytes verbatim.** No `.trim()`, no `JSON.parse`, no
   decode-then-re-encode of the body. Read STDIN as a `Uint8Array`,
   decode UTF-8 once, pass to `signWebhookPayload({ body })`.
4. **No new `eslint-disable` / `@ts-ignore` / `@ts-expect-error` /
   `as any` under `packages/cli/**`.** Boundary casts at
   `process.stdin` (the `as unknown as StdinLike` shape from 0106)
   are acceptable — copy the pattern from
   `packages/cli/src/commands/webhook-verify.ts:179`.
5. **Reuse, don't fork.** Import from `webhook-verify.ts` if you need
   to extract the stdin/body-reading helper into a shared module
   (a new `packages/cli/src/commands/webhook-body.ts` is in scope as
   a 5th path IF you actually de-duplicate; otherwise keep the
   helper inline like 0106 did).
6. **Implementer report on PR branch.** Commit
   `ai/reports/task-0107-implementer.md` to the PR branch in the
   SAME commit (or a follow-up commit before opening the PR). Do
   NOT repeat the 0106 gap.

## Test coverage (≥ 12 cases)

`packages/cli/src/__tests__/webhook-sign.test.ts` MUST cover at
minimum:

1. Happy path human mode — body via STDIN, valid `--timestamp`, exit 0,
   stdout matches `^signature: sha256=[0-9a-f]{64}\ntimestamp: \d+\n$`.
2. Happy path json mode — exit 0, stdout is single-line valid JSON
   with `signature` and `timestamp` string fields.
3. Body via `--body=PATH` reads file bytes binary-safe.
4. `--body=PATH` + piped STDIN → `UsageError`, exit 2.
5. Missing body (no flag, no STDIN) → `UsageError`, exit 2.
6. Missing `--secret` → `UsageError`, exit 2.
7. Missing `--timestamp` → `UsageError`, exit 2.
8. `--timestamp=abc` (not an integer) → `UsageError`, exit 2.
9. `--timestamp=-5` (negative) → `UsageError`, exit 2.
10. Multi-byte UTF-8 body bytes signed correctly (compose body with
    `Buffer.from("héllo 漢字", "utf8")` → assert deterministic
    signature against a precomputed value).
11. `--output=invalid` → `UsageError`, exit 2.
12. **Round-trip integration** — generate a signature with
    `webhook sign`, pass it back through the helper's
    `verifyWebhookSignature` (NOT through the CLI — direct helper
    import) using the same body/timestamp/secret, assert `ok: true`.

You can test commands via the existing
`packages/cli/src/cli-runner.ts` `runCli` factory with stdin/stdout
injection — copy the harness from
`packages/cli/src/__tests__/webhook-verify.test.ts`.
