# Required Outcomes

1. Add public audit response contracts.
   - Add a public audit entry projection type, for example
     `PublicAuditEntry`, and `ListAuditEntriesResponse`.
   - Public audit response IDs must use public boundary IDs where the current
     platform can map them:
     - `org_` for organizations;
     - `prj_` for projects;
     - `env_` for environments;
     - `inv_` for invitations;
     - `mem_` for members.
   - Do not expose raw database UUIDs for known current audit subject kinds or
     scope columns.
   - Preserve useful fields: event ID, category, event type, source, actor,
     subject, description, occurredAt, requestId, correlationId, scope IDs, and
     safe payload.
   - Redact response payload fields named by `redactPaths` before returning
     audit entries. If a redact path cannot be applied safely, omit or replace
     the targeted value rather than leaking it.

2. Add `audit.read` policy semantics.
   - Add `audit.read` to the shared policy contract and policy engine.
   - Treat it as organization-scoped.
   - Role semantics:
     - organization `owner` and `admin` may read the organization audit log;
     - organization `builder`, `viewer`, and `billing_admin` may not for V1;
     - project-scoped roles alone must not grant organization-wide audit read.
   - Unknown or malformed facts must not widen access.
   - If implementation finds a strong existing spec reason to allow viewers,
     write a proposal instead of widening access silently.

3. Fix `queryAuditByOrg` category filtering.
   - Extend the events repository query params or method shape so
     `queryAuditByOrg` can accept optional `category`.
   - When `category` is provided, SQL must filter by `org_id` and `category`
     while preserving cursor pagination by `(occurred_at DESC, id DESC)`.
   - Keep SQL parameterized.
   - Add db tests proving category filtering, cursor filtering, ordering, and
     safe internal-error mapping.

4. Add `apps/events-worker`.
   - Scaffold a Cloudflare Worker consistent with existing Worker patterns.
   - Include `/health`.
   - Add route:
     `GET /v1/organizations/{orgId}/audit`.
   - Parse `org_` public IDs and return 404 for malformed IDs.
   - Require actor headers from api-edge; missing actor returns 401.
   - Require `SOURCEPLANE_DB`, `MEMBERSHIP_WORKER`, and `POLICY_WORKER`;
     missing bindings return safe 503.
   - Use membership-worker authorization-context with actor subject and raw org
     UUID before policy-worker.
   - Use policy-worker action `audit.read` with resource
     `{ kind: "organization", orgId }`.
   - Policy denial and membership-context failure must return 404 to prevent
     organization enumeration.
   - Use `createEventsRepository(...).queryAuditByOrg(...)`.
   - Support query params:
     - `limit` default 50, max 100;
     - `cursor` opaque and endpoint-owned;
     - optional `category` with a conservative validation rule such as
       lowercase letters, numbers, underscore, hyphen, and dot.
   - Return success envelope:
     `{ data: { auditEntries: PublicAuditEntry[] }, meta: { requestId, cursor } }`.

5. Add api-edge audit facade.
   - Add `EVENTS_WORKER` to api-edge env typing and stage/prod wrangler service
     bindings.
   - Forward only `GET /v1/organizations/{orgId}/audit` to `EVENTS_WORKER`
     after identity resolution.
   - Preserve query params.
   - Forward actor headers, request ID, traceparent, and idempotency-key as
     appropriate.
   - Continue not forwarding bearer tokens.
   - Do not forward a body for GET.
   - Missing `EVENTS_WORKER` must return a safe 503.
   - Do not expose target-specific audit routes or security-event routes.

6. Add Worker deployment/component wiring.
   - Add `apps/events-worker/package.json`, `tsconfig.json`, `wrangler.jsonc`,
     and `component.yaml` using existing worker patterns.
   - Stage/prod events-worker must be private: `workers_dev: false`.
   - Stage/prod events-worker must bind:
     - `SOURCEPLANE_DB` to the existing verified Hyperdrive IDs;
     - `MEMBERSHIP_WORKER` to same-environment membership-worker;
     - `POLICY_WORKER` to same-environment policy-worker.
   - api-edge stage/prod must bind `EVENTS_WORKER` to same-environment
     events-worker.
   - No Terraform or migration changes are expected.

7. Add focused tests.
   - contracts tests for the public audit response shape.
   - policy-engine tests for `audit.read` role matrix and project-role denial.
   - db tests for category-aware `queryAuditByOrg`.
   - events-worker tests for:
     - successful list with pagination and category;
     - malformed org ID 404;
     - missing actor 401;
     - missing bindings 503;
     - invalid limit/cursor/category 422;
     - membership-context failure 404;
     - policy denial 404;
     - repository failure 503;
     - public ID mapping and payload redaction.
   - api-edge tests for:
     - GET audit forwarding with query params;
     - actor/request/trace/idempotency header forwarding;
     - bearer token not forwarded;
     - no body forwarded;
     - missing `EVENTS_WORKER` 503;
     - non-GET method 405;
     - existing org/project routes remain intact.
