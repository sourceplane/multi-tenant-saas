## Objective

Add a new CLI subcommand `sourceplane webhook verify` that reads a
webhook payload + signature headers (from flags or STDIN) and uses
`@saas/webhook-verifier` to verify the HMAC-SHA256 signature. Exit
code 0 on a valid signature, exit code 4 on any verifier failure
(reusing the existing `formatCliError` exit-code semantics).

### Surface (behaviour spec)

```
sourceplane webhook verify
  --secret=SECRET                        REQUIRED — webhook signing secret
  --signature=HEADER_VALUE               REQUIRED — value of the
                                         `X-Webhook-Signature` header,
                                         e.g. `sha256=abcd...`
  --timestamp=HEADER_VALUE               REQUIRED — value of the
                                         `X-Webhook-Timestamp` header,
                                         a unix-epoch-seconds integer
  --body=PATH                            optional — read body from this
                                         file path; mutually exclusive
                                         with STDIN
  --tolerance-seconds=N                  optional — replay tolerance;
                                         default 300 (matches helper)
  --output=human|json                    optional — default human

If `--body` is omitted, the CLI reads STDIN until EOF and treats those
bytes verbatim as the webhook body.
```

#### Output

- Human mode (success):
  ```
  ok: true
  reason: <empty>
  ```
  Single trailing newline. Exit code 0.
- Human mode (failure):
  ```
  ok: false
  reason: <reason-code>
  ```
  Exit code 4. Reason codes are exactly the strings the helper
  returns (`missing_signature_header`, `missing_timestamp_header`,
  `malformed_signature`, `malformed_timestamp`,
  `timestamp_out_of_tolerance`, `signature_mismatch`, plus any others
  the helper exports — read `packages/webhook-verifier/src/index.ts`
  for the canonical list and surface them verbatim).
- JSON mode (success): `{"ok": true}` on a single line, exit 0.
- JSON mode (failure): `{"ok": false, "reason": "<code>"}` on a
  single line, exit 4.
- All output goes to **stdout** (not stderr) — verifier failure is
  not a CLI error, it is the command's normal result. Exit code
  carries the signal.

#### Usage / argument errors

- Missing required flag (`--secret`, `--signature`, `--timestamp`):
  throw `UsageError` with a message naming the missing flag → exit 2
  via `formatCliError`. Do NOT use exit 4 for argument-shape
  failures.
- `--body=PATH` and STDIN both readable (i.e. `--body` provided AND
  STDIN is a TTY-less pipe with bytes): throw `UsageError` →
  exit 2.
- `--tolerance-seconds=N` not a non-negative integer:
  `UsageError` → exit 2.
- `--body=PATH` but file does not exist: throw `UsageError` →
  exit 2 with a message that includes the path.

## Hard rules (no exceptions)

- **Zero edits** to: `apps/**` (any worker), `packages/sdk/**`,
  `packages/contracts/**`, `packages/webhook-verifier/**`
  (the helper just landed; do not modify it),
  `apps/web-console-next/**`, `tooling/**`, `tests/api-edge/**`,
  `kiox.lock`.
- The new command must NOT call `Sourceplane`, the api-edge, or any
  network. No `client.*`, no `fetch(`, no `/v1/`. Local crypto only.
- The new command must NOT require auth, an active org, or a
  context store. It does not call `resolveOrgId`. Tests must
  exercise the command with a default `CommandContext` that has no
  bearer token loaded.
- No `node:crypto`, no `node:buffer` polyfills, no Node-only
  imports. The verify path must run through the helper which is
  WebCrypto-only.
- Body reading must be **binary-safe**: read the raw bytes (or
  `Buffer` → `Uint8Array`); do NOT JSON-parse or `.trim()` or
  decode-then-re-encode the body. The verifier is byte-exact.
- For STDIN reading, read until EOF using async iteration over
  `process.stdin`. Decode as UTF-8 only at the very last step
  before passing to the helper if the helper requires a string;
  preferably pass `Uint8Array` directly. Inspect the helper's
  `verifyWebhookSignature` input type and match it.
- No new `eslint-disable`, `@ts-ignore`, `@ts-expect-error`,
  `as unknown as`, or `as any` anywhere under
  `packages/cli/**`.
- Reuse existing `UsageError` from `packages/cli/src/errors.ts` for
  argument errors. Reuse `formatOutput` from
  `packages/cli/src/output/index.ts` if it cleanly fits the two-line
  human surface; otherwise emit two `ctx.stdout(...)` calls
  directly.
