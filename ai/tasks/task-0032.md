# Objective

Add the public project-list endpoint:

`GET /v1/organizations/{orgId}/projects`

This task must add a bounded `project.list` policy action and route the list
through `api-edge -> projects-worker -> membership-worker -> policy-worker`,
using cursor pagination and the existing projects repository.

# Required Outcomes

1. Add `project.list` to the policy contract.
   - Add `project.list` to `ORGANIZATION_ACTIONS` / known action coverage.
   - Treat `project.list` as organization-scoped, not project-scoped.
   - Organization roles `owner`, `admin`, `builder`, and `viewer` may list
     projects in their organization.
   - `billing_admin` must not gain project-list access.
   - Project-scoped roles alone must not authorize organization-wide listing;
     a future filtered "my projects" list can be a separate task.
   - Keep `project.read` requiring explicit `projectId`.

2. Implement projects-worker list behavior.
   - Add `GET /v1/organizations/{orgId}/projects`.
   - Parse `orgId` from the public `org_` ID and return 404 for malformed IDs.
   - Resolve the actor from api-edge forwarded headers, same as create/get.
   - Parse `limit` and `cursor` query parameters using the established cursor
     contract: default 50, max 100, opaque versioned cursor, invalid params
     return `validation_failed`.
   - Do not import from `apps/membership-worker`; add a local helper or a small
     shared helper only if it stays narrowly scoped and matches existing
     conventions.
   - Call membership-worker authorization-context using actor subject and raw
     org UUID.
   - Call policy-worker authorize with action `project.list` and resource
     `{ kind: "organization", orgId }` or an equivalent organization-scoped
     resource shape.
   - Fail closed: membership-context failures, malformed envelopes, policy
     errors, malformed policy responses, or policy denial return safe 404 or
     service-unavailable style errors consistent with Task 0031 behavior and
     must not leak internals.
   - Use `listProjectsPaged(orgId, pageParams)`; never query projects across
     organizations.
   - Return `200` with `{ projects: PublicProject[] }` and `meta.cursor`.
   - Public responses must use `org_` and `prj_` IDs, never raw UUIDs.

3. Update api-edge project facade.
   - Forward `GET /v1/organizations/{orgId}/projects` to `PROJECTS_WORKER`
     after identity resolution.
   - Preserve query string parameters (`limit`, `cursor`).
   - Continue to forward actor headers and request/trace/idempotency headers as
     appropriate.
   - Continue not forwarding bearer tokens.
   - Keep `POST /v1/organizations/{orgId}/projects` and
     `GET /v1/organizations/{orgId}/projects/{projectId}` behavior intact.
   - Do not expose any project update/archive/delete or environment route.

4. Keep bounded-context seams clean.
   - `projects-worker` must not import membership repository code.
   - `projects-worker` must not query `membership.*` storage directly.
   - Policy changes must be limited to the `project.list` action and role
     matrix needed for this endpoint.
   - No public policy or membership internal route may be exposed through
     `api-edge`.

5. Add tests.
   - Contracts tests must cover `project.list` as a known policy action and
     preserve `ListProjectsResponse`.
   - Policy-engine tests must cover:
     - owner/admin/builder/viewer allow `project.list`;
     - billing_admin denies `project.list`;
     - no memberships deny;
     - cross-org facts deny;
     - project-scoped roles alone do not grant org-wide list;
     - `project.read` still requires explicit `projectId`.
   - projects-worker tests must cover:
     - list success with authorization-context + policy allow;
     - pagination defaults, limit bounds, invalid cursor, next cursor;
     - malformed org ID returns 404;
     - missing DB, membership binding, or policy binding fails safely;
     - membership-context failure/malformed envelope fails closed;
     - policy denial and policy fetch/malformed envelope fail closed;
     - repository failure returns safe error;
     - raw UUIDs are not exposed in list responses.
   - api-edge tests must cover:
     - GET collection route forwards to `PROJECTS_WORKER` after identity
       resolution;
     - query params are preserved;
     - actor headers are forwarded;
     - bearer token is not forwarded;
     - missing `PROJECTS_WORKER` returns safe 503.

6. Preserve Orun/component behavior.
   - Local Orun validation must pass.
