## Architect Brief

- **Product bar**: **Stripe SDK quality** for the base client and resource
  shape. Idempotency-Key surfaced as a first-class request option; typed
  errors that map 1:1 to `ERROR_CODES` from `@saas/contracts/errors`;
  `RateLimitError` carries the `Retry-After` and `X-RateLimit-*` headers Task
  0097 will start emitting; pagination/result shapes match the on-wire JSON
  emitted by api-edge today.
- **User moment this PR improves**: an external integrator (or our future
  CLI / console refactor) can `import { Sourceplane } from '@saas/sdk'` and
  call `client.projects.create({ name }, { idempotencyKey, signal })` with
  full TypeScript inference, instead of hand-rolling fetch + envelope parsing.
- **Latitude (no spec proposal needed)**: HTTP transport implementation
  (native `fetch` recommended, but free to wrap), error class hierarchy
  shape, retry policy on 429/5xx (token-bucket-aware backoff is a plus but
  not required), pagination iterator design, namespace layout under
  `client.<resource>.<action>`, internal file structure, request-id
  propagation pattern, telemetry hook seam.
- **Hard rules (require spec proposal to deviate)**: do NOT introduce new
  request/response types — re-export from `@saas/contracts`. Do NOT change
  any contract type. Do NOT consume Cloudflare-specific globals (sdk must
  run in browser, Node ≥20, Workers, Bun). Do NOT bundle a transport
  adapter that imports `apps/**` or any worker source. Do NOT introduce a
  new public route or talk to anything other than the `api-edge` base URL
  the caller passes. No secrets in fixtures or tests.
- **Failure modes that invalidate the PR even if tests pass**: caller
  receives a raw `Response` instead of a typed envelope on error; `error`
  branch leaks an `unknown` rather than a discriminated union keyed on
  `ERROR_CODES`; `idempotencyKey` accepted but not actually sent on the
  wire; bundle pulls in Node-only APIs (`node:*`, `crypto.randomUUID` from
  the wrong runtime, etc.) breaking browser/Workers; sdk depends on
  `@saas/contracts` only via `dist/` instead of the source `exports`
  surface; new `eslint-disable*`, `@ts-ignore`, `@ts-expect-error`, or
  `as unknown as` introduced (Track A/B hazard ban still in force).
- **Recommended stack**: native `fetch`, no runtime deps. Zero transitive
  dependencies in `dependencies` is a strong default; `devDependencies`
  may include `vitest`, `@types/node`, and the workspace `tsconfig` /
  `eslint` configs. Document any deviation with a one-line rationale in
  the implementer report.

## Objective

Stand up the new `packages/sdk` workspace as a typed, dependency-free,
runtime-agnostic SaaS client. Ship the base transport (configurable base
URL, auth, idempotency-key, request-id, abort signal), the typed error
hierarchy keyed on `ERROR_CODES`, and **two pilot resource clients** that
hit existing api-edge surfaces end-to-end: `client.organizations.*` (list
/ get / create) and `client.projects.*` (list / get / create / archive).
This lays the contract for the rest of the resource clients (Task 0099+
will fan out without further architecture decisions) and unblocks
`packages/cli` (B4 second half).

## PR Boundary

In scope:
1. New workspace `packages/sdk/` with `package.json`, `tsconfig.json`,
   `tsconfig.build.json`, `eslint.config.js`, `src/`, `src/__tests__/` (or
   colocated `*.test.ts` per repo convention — match what `packages/contracts`
   uses).
2. `src/index.ts` exports `Sourceplane` (the client class), every resource
   namespace surface, the typed error hierarchy, and re-exports the
   request/response types from `@saas/contracts` so consumers don't import
   contracts directly.
3. Base transport in `src/transport.ts` (or equivalent): configurable
   `baseUrl`, optional `auth: { kind: "bearer"; token: string } | { kind:
   "session"; cookie: string }`, optional `defaultHeaders`, per-request
   `idempotencyKey`, `signal`, `requestId` (auto-generated when omitted).
4. Error decoder: parses the api-edge envelope (`{ error, message,
   requestId?, fields? }`) into a discriminated `SourceplaneError` union.
   `RateLimitError` exposes `retryAfterSeconds`, `limit`, `remaining`,
   `resetAt`, scope (`org` | `identity`) — sourced from the headers Task 0097
   will emit. SDK does not depend on Task 0097 having merged; it just exposes
   the field shape so headers will flow through cleanly the moment they
   start arriving.
5. Two pilot resource clients:
   - `OrganizationsClient` — list / get / create (POST goes through the
     idempotency-key surface).
   - `ProjectsClient` — list (org-scoped) / get / create (POST with
     idempotency-key) / archive.
6. Vitest unit tests against a stub `fetch` covering: success path,
   error-envelope decode for every `ERROR_CODES` value the pilot clients
   can return, idempotency-key propagation on POST, abort-signal propagation,
   request-id auto-generation when caller omits, request-id passthrough when
   caller provides, rate-limit header decoding (synthetic 429 response shape
   matching Task 0097's planned envelope), JSON parse fallback when the
   server returns a non-JSON 5xx body. Aim for ≥ 90% statement coverage on
   `transport.ts` and the error decoder.
7. README.md inside `packages/sdk/` documenting: install (workspace), one
   getting-started snippet, error handling pattern, idempotency snippet,
   environment compatibility matrix (browser / Node ≥ 20 / Workers / Bun).
8. The new package shows up in `pnpm install`, passes `pnpm --filter
   @saas/sdk typecheck`, `pnpm --filter @saas/sdk lint`, and `pnpm --filter
   @saas/sdk test` cleanly.
9. **No** `component.yaml` for now (sdk is a library, not a deployable). If
   the orun plan complains, the implementer adds the minimum it needs to
   stay green and records the rationale; do not add a component just to be
   thorough.

## Constraints

1. Runtime-agnostic: source must work on browser, Node ≥ 20, Cloudflare
   Workers, and Bun without conditional imports. Implementer documents the
   compatibility matrix in the README.
2. Zero runtime dependencies preferred. If the implementer adds a runtime
   dep, the report records the one-line rationale and confirms the dep
   ships ESM and runs in all four target runtimes.
3. The error hierarchy MUST cover every `ERROR_CODE` value reachable from
   the orgs/projects facades. Unknown codes decode to a generic
   `SourceplaneError` carrying the raw envelope (forward-compatible).
4. `RateLimitError` field names match the headers Task 0097 will emit (per
   `specs/contracts/api-guidelines.md` line 127 and the Task 0097 prompt).
   The sdk decodes them defensively — missing headers produce `null`/
   `undefined`, never throw.
5. `idempotencyKey` is a per-request option on every POST surface. When the
   caller omits it on a POST, the sdk does NOT auto-generate one — the
   server's contract makes the key opaque and caller-owned (Stripe parity).
6. `requestId` is auto-generated (`crypto.randomUUID()` from the platform's
   global, not `node:crypto`) when the caller omits it; passed through
   verbatim when provided.
7. Hazard ban (carried from Tasks 0096b–f and 0097): no new
   `eslint-disable*`, `@ts-ignore`, `@ts-expect-error`, `as unknown as`,
   `as any`. Use real types or fix the contract.
8. The sdk MUST consume types from `@saas/contracts` via the same `exports`
   subpaths the rest of the repo uses (e.g.
   `import { ErrorCode } from "@saas/contracts/errors"`). Do not reach into
   `@saas/contracts/src/**` or `dist/**`.
9. No fixtures, snapshots, or tests may contain a real bearer token, API
   key, or signing secret. Use literal placeholder strings (`"test-token"`).
