## Objective

Bump the four deprecated GitHub Actions in `.github/workflows/ci.yml` to the
latest stable major that ships the Node 24 runtime, eliminating the Node 20
deprecation annotation before the **June 16 2026** hard cutover. No behaviour
change to the plan/run pipeline.

## Architect Brief

- Product bar: **Cloudflare Workers / GitHub Actions** CI hygiene — the pipeline
  must stay byte-for-byte equivalent in behaviour; only the action runtimes move.
- User moment improved: maintainers stop seeing the deprecation banner on every
  run, and the pipeline is de-risked against the forced June 16 cutover.
- Free to decide: whether to pin floating majors (`@v6`/`@v7`/`@v8`/`@v4`) or
  exact patch tags (`@v6.0.2`, etc.). Recommended: floating majors for these
  first-party/well-known actions (matches the existing `@v4`/`@v3` style and
  auto-absorbs patch fixes). Record the choice with one-line rationale.
- Must NOT decide without a spec proposal: bumping or repinning
  `sourceplane/orun-action`, changing job structure, permissions, the matrix
  strategy, env wiring, or any `orun plan`/`orun run` argument.
- Failure modes that invalidate the PR even if green: any change to the two
  `orun` invocations or their args/env; an artifact upload/download major
  mismatch that breaks plan.json hand-off between the `plan` and `run` jobs;
  touching `intent.yaml` or any component.

The implementer has full latitude on the floating-major-vs-exact-tag pin style
inside the Constraints below. Decisions taken under this latitude must be
recorded with one-line rationale in the implementer report. Anything outside
that requires a spec proposal before implementation.
