# Objective

Add the first policy-gated member administration read endpoint:

`GET /v1/organizations/{orgId}/members`

The endpoint should be available through both `membership-worker` and the
public `api-edge` facade, authorize through `policy-worker` using action
`organization.member.list`, and return safe member records for the organization.

# PR Boundary

api-edge facade routing, tests, and the implementer report.

This PR must not add member removal, role updates, invitation create/list/revoke
routes, invitation acceptance, email delivery, audit/event persistence, database
migrations, policy-engine behavior changes, Terraform/resource changes, or UI
work.

# Required Outcomes

1. Add shared member-list contract types.
   - Extend `packages/contracts/src/membership.ts` with public response types
     for listing organization members.
   - Include member public ID, subject type, subject ID, status, joined/created
     timestamp, and organization-scoped role names.
   - Do not expose raw member UUIDs, role-assignment UUIDs, database internals,
     token hashes, or fields owned by identity such as email/display name.
   - If the implementation sees project-scoped role assignments, do not expose
     raw project UUIDs. Either omit unsupported scope details or return a safe
     scope shape that cannot leak raw database IDs.

2. Add safe member public IDs.
   - Add `mem_` public ID formatting in `apps/membership-worker/src/ids.ts` for
     member UUIDs.
   - No parse helper is required unless the implementation needs one for tests.
   - Keep existing `org_` behavior unchanged.

3. Add membership-worker member-list behavior.
   - Add `GET /v1/organizations/{orgId}/members`.
   - Parse the `orgId` path segment with the existing org public ID helper.
   - Require actor headers, `SOURCEPLANE_DB`, and `POLICY_WORKER`.
   - Load the actor's role assignments for the organization and authorize
     through `POLICY_WORKER` with action `organization.member.list`.
   - Send policy resource `{ kind: "organization", id: orgUuid, orgId: orgUuid }`.
   - Fail closed when role lookup, policy binding, policy fetch, malformed policy
     response, or policy denial occurs.
   - For policy denial, preserve no-enumeration behavior and return `not_found`.
   - For infrastructure failures, return a safe `internal_error` or
     service-unavailable style envelope without leaking internals.
   - Use `repo.listMembers(orgUuid)` for active members.
   - Attach organization-scoped roles for each listed member using existing
     repository methods. If any role lookup fails, return a safe
     `internal_error` and do not return partial member data.
   - Keep `POST /v1/organizations`, `GET /v1/organizations`, and
     `GET /v1/organizations/{orgId}` behavior unchanged except for shared helper
     extraction if needed.

4. Add api-edge facade routing.
   - Make `api-edge` recognize and forward
     `GET /v1/organizations/{orgId}/members` to `MEMBERSHIP_WORKER`.
   - Preserve auth resolution through `IDENTITY_WORKER`.
   - Continue forwarding actor headers, request ID, traceparent, and allowed
     request headers only.
   - Do not forward raw bearer tokens to `membership-worker`.
   - Reject unsupported methods for the members route with the existing safe
     error envelope.

5. Add or update tests.
   - Membership-worker tests must cover:
     - allowed member list with expected response shape;
     - policy request action/resource for `organization.member.list`;
     - policy denial returns `not_found`;
     - missing policy binding fails closed;
     - actor role-list failure fails closed;
     - member role-list failure returns safe `internal_error`;
     - raw member UUIDs and role-assignment UUIDs are not exposed;
     - project-scoped role assignments do not leak raw project UUIDs.
   - Api-edge tests must cover:
     - route matching for `/v1/organizations/{orgId}/members`;
     - only `GET` is allowed;
     - auth resolution and actor header forwarding;
     - no raw bearer token forwarding;
     - downstream success/error passthrough for the new route.
