## Objective

Ship roadmap **B9 — Entitlement-decision observability**: emit **counts only** (no provider
payloads, no secrets) for entitlement decisions, keyed by **caller × entitlement key ×
outcome**, from `billing-worker`, persist them in a structured, queryable projection, and
surface a **narrow read-only aggregation endpoint via `admin-worker`** so a support/on-call
operator can see who is hitting the gate. Internal-only; deny-by-default reads; no public
exposure.

## Milestone

**B9 — Entitlement-decision observability (counts by caller × entitlement key × outcome).**

Deliver, end to end:

1. **Decision emission (`billing-worker`)** — when `decideEntitlement` produces a decision
   (allowed or denied), record a structured **decision observation** (orgId, entitlementKey,
   outcome = `allowed` | `denied`, denial reason when denied, occurredAt). Counts/observations
   only — **never** emit limit values, subscription IDs, provider payloads, or secrets. Emit
   on the `check-entitlement` path only (the existing internal decision route). Make the
   emission **best-effort and non-blocking**: a recording failure must NOT change the
   entitlement decision returned to the caller (mirror the admin-worker denial-audit
   best-effort pattern).
2. **DB persistence** — a new checked-in migration `packages/db/src/migrations/150_*` (next
   ordinal after `140_support_action_records`) creating the entitlement-decision observation
   storage this feature owns (forward-only, idempotent `IF NOT EXISTS` guards, no backfill).
   Decide the cleanest shape for **count aggregation by (orgId, entitlementKey, outcome)** over
   a time window — either an append-only observation table aggregated at read time, or a
   rollup counter; pick one, justify it in the report, and follow peer migration conventions.
   Register the migration + any new bounded context in `manifest.ts` / `types.ts`.
3. **Admin read surface (`admin-worker`)** — a new internal-only handler + route (e.g.
   `GET /v1/internal/support/organizations/:orgId/entitlement-decisions`) returning a **narrow
   aggregation**: per `(entitlementKey, outcome)` counts (and optionally denial-reason
   breakdown) over a bounded window. Reuse the existing **deny-by-default
   `authorizeSupportAction`** gate (a denied read emits `support.access_denied`); reuse the
   narrow-projection discipline (no limit values, no subscription IDs, no secrets). Keyset or
   simple bounded result; no unbounded scans.
4. **Tests** — handler-level tests for: decision-observation emission on both allowed and
   denied paths (and that an emission failure does not change the entitlement response);
   deny-by-default on the admin read (+ `support.access_denied`); the aggregation read
   returning correct per-key/outcome counts with no secret fields. Mirror the
   admin-worker + billing-worker handler test style (Jest + ts-jest, injected-deps seam).

## Constraints

1. **Decision integrity**: the entitlement response is frozen; the observation is a pure
   side-effect and must be best-effort + non-blocking (a recording failure is swallowed; the
   decision still returns unchanged).
2. **Counts only, secret-free**: observations carry orgId + entitlementKey + outcome
   (+ denial reason) + timestamp — never limit values, subscription IDs, payloads, or secrets.
3. **Deny-by-default reads**: the admin aggregation read reuses `authorizeSupportAction`; an
   unauthorized read fails closed and emits `support.access_denied`.
4. **Internal-only**: neither the billing emission nor the admin read is reachable via
   api-edge; no public route.
5. **Bounded reads**: aggregation is over a bounded window / bounded result set — no unbounded
   table scans exposed through the API.
6. **Orun/golden-path**: no new component is required (extends existing workers + db), but if
   a new `tests/*` package is added it MUST ship `component.yaml`. Migration is forward-only,
   `NNN_name` ordinal, checked in.
7. Terraform backend / secrets conventions unchanged; do not log secrets.
