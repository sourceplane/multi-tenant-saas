## Objective

Ship the B7 **audit-log filtering + export** milestone end to end: take the
existing org-scoped audit stream from "category-only" to buyer-credible by
adding filters for **actor** (actorId / actorType), **resource/subject**
(subjectKind / subjectId), **action** (eventType), and **time range**
(occurredAt `from`/`to`), plus an **NDJSON export** of the filtered stream,
across DB → worker → contracts → SDK → CLI → Console. Satisfies
`specs/roadmap.md` §B7 and `specs/components/09-events-audit-observability.md`
("Audit history is queryable by organization and target resource"; required
audit coverage spans org/member/project/key/webhook/billing/config events that
buyers expect to slice by who/what/when).

## Architect Brief

The backend filter primitive is the load-bearing piece; every consumer is a
thin pass-through on top of it. Build bottom-up:

1. **DB** (`repository.ts` `queryAuditByOrg`): add OPTIONAL, independently
   combinable WHERE clauses — `actor_id`, `actor_type`, `subject_kind`,
   `subject_id`, `event_type`, and a time window on `occurred_at`
   (`occurred_at >= $from` / `occurred_at <= $to`). Preserve the legacy
   `org_id IN ($1,$2)` dual-format and the `ORDER BY occurred_at DESC, id DESC`
   keyset cursor ordering EXACTLY — filters narrow the set, they must not change
   ordering or break cursor stability. Use parameterized placeholders only
   (no string interpolation). Mirror `queryAuditByTarget`'s clause-builder style.
2. **Worker** (`pagination.ts` + `handlers/list-audit.ts`): parse + validate the
   new params. `from`/`to` MUST validate against `ISO_TS_RE` (reject malformed
   timestamps with 400, same as existing cursor validation). `actorType` should
   validate against the `EventActorType` set; `actorId`/`subjectId`/`subjectKind`
   /`eventType` against a conservative regex (reuse/extend the `CATEGORY_RE`
   style — bounded length, safe charset). Thread validated params into the repo
   call. Unknown/empty params are ignored, not errors. Keep `redactPayload`
   behaviour unchanged.
3. **Contracts** (`events.ts`): extend `AuditQueryByOrg` with the optional
   filter fields (`actorId?`, `actorType?`, `subjectKind?`, `subjectId?`,
   `eventType?`, `from?`, `to?`). Do NOT widen `PublicAuditEntry`, the response
   envelope, or the org-less-audit prohibition (spec §"Identity Security
   History" — org_id is always required; do not add a cross-org query path).
4. **SDK** (`events.ts`): add the same optional fields to the `by:"org"` arm of
   `ListAuditEntriesQuery` and thread them through `buildAuditRequest` (and the
   `iterAuditEntries` per-page query reconstruction so filters survive
   pagination). Add a buyer-facing **NDJSON export helper** (e.g.
   `exportAuditEntriesNdjson(orgId, query)`) layered over `iterAuditEntries`
   that yields/returns newline-delimited JSON of each `PublicAuditEntry`. Keep
   the `meta.cursor ?? null` envelope read pattern — never read cursor from body.
5. **CLI** (`cross-reads.ts` + `cli-runner.ts`): add flags `--actor` (maps to
   actorId), `--actor-type`, `--subject-kind`, `--subject-id`, `--event-type`,
   `--from`, `--to` to `audit list`, all composable with existing `--category`
   /`--limit`/`--cursor`/`--all`/`--json`. Add an export mode (e.g.
   `--format=ndjson` or an `audit export` subcommand) that streams the filtered
   stream as NDJSON to stdout via the SDK export helper. Update the usage string.
6. **Console** (`audit/page.tsx`): add a filter UI (actor, subject, action/
   eventType, from/to date-time inputs) wired through the SDK `wrap()` only —
   ZERO `fetch`. Add Load-more cursor pagination (the page currently does a
   single fetch — wire `meta.cursor`). Add an "Export NDJSON" action that
   downloads the filtered stream. Honour the U4/U8 design bars: designed empty
   state + skeleton/loading state for the list and the export action.
7. **Tests** on every surface: DB (filter clauses + cursor stability under
   filters), worker (param validation: good + malformed `from`/`to`, actorType
   rejection, composition), contracts (type shape), SDK (buildAuditRequest query
   mapping + iterator filter survival + NDJSON helper), CLI (flag parsing,
   export output, mutual-exclusion rules), Console harness
   (web-console-next-tests: filter wiring, Load-more, export button states).
