## Objective

Close the two SDK-side gaps surfaced by Task 0101, then re-wire the CLI
so the two `transport.*` workaround sites disappear. After this task,
**every** CLI command is dispatched through a typed `@saas/sdk` resource
client; `transport.request` and `transport.fetchImpl` are no longer
called from anywhere under `packages/cli/src/commands/**`. This closes
Track B4.

## PR Boundary

In scope (one PR):

1. **Add `EnvironmentsClient` to `@saas/sdk`** — new file
   `packages/sdk/src/environments.ts` mirroring the shape of
   `packages/sdk/src/projects.ts`:

   ```ts
   class EnvironmentsClient {
     list(orgId: string, projectId: string,
          opts?: RequestOptions): Promise<ListEnvironmentsResponse>;
     get(orgId: string, projectId: string, envId: string,
         opts?: RequestOptions): Promise<GetEnvironmentResponse>;
     create(orgId: string, projectId: string, body: CreateEnvironmentRequest,
            opts?: RequestOptions): Promise<CreateEnvironmentResponse>;
     archive(orgId: string, projectId: string, envId: string,
             opts?: RequestOptions): Promise<ArchiveEnvironmentResponse>;
   }
   ```

   - Consume types directly from `@saas/contracts` (no contract edits).
   - `encodeURIComponent` on every dynamic segment.
   - Caller-owned `RequestOptions.idempotencyKey` on `create` and
     `archive`. SDK MUST NOT generate a key.
   - Wire on `Sourceplane` as `client.environments` in
     `packages/sdk/src/index.ts`.

2. **Surface paginated audit reads at the SDK boundary** — extend
   `packages/sdk/src/events.ts` with an async iterator (Option 1 in the
   proposal):

   ```ts
   class EventsClient {
     iterAuditEntries(
       orgId: string,
       query?: ListAuditEntriesQuery,
       opts?: RequestOptions,
     ): AsyncIterable<PublicAuditEntry>;
   }
   ```

   - Internal loop on `meta.cursor` until `null` / `undefined`.
   - Apply the same loop guards the CLI shipped: max-page cap (1000) and
     a `seenCursors` Set, so a misbehaving server cannot infinite-loop.
   - Do NOT change the existing `listAuditEntries(...)` shape (single-
     page primitive remains; the iterator composes on top of it).
   - If `Transport.request` does not already expose a way to read
     `meta.cursor`, add a typed envelope-aware helper inside
     `transport.ts` (e.g. `requestWithEnvelope<T>(): Promise<{ data: T;
     meta: { requestId: string; cursor?: string | null } }>`) so the
     iterator does not have to reach into raw `fetchImpl`. Implementer
     latitude on the exact shape; document the choice in the report.

3. **SDK tests** — extend `packages/sdk/src/__tests__/`:
   - `environments.test.ts` (new) covering list/get/create/archive happy
     paths, `encodeURIComponent` on all segments, idempotency-key
     verbatim passthrough on create/archive, `SourceplaneError`
     propagation with request-ID.
   - `events.test.ts` (extend) covering the new iterator: multi-page
     consumption, single-page termination on `cursor === null`,

     propagation.
   - Hazard scan target on the SDK: 0 hits

4. **CLI re-wiring** — drop the two `transport.*` workaround sites:
