## Objective

Wire `apps/membership-worker` as the next bounded-context consumer of the billing entitlement decision seam by gating member invitations on `limit.members`. This proves the internal billing entitlement seam supports more than one caller while preserving membership/billing bounded-context separation and the service-binding-only caller identity contract.

## PR Boundary

1. Add a small internal billing entitlement client in `apps/membership-worker` for `POST /v1/internal/billing/entitlements/check`, using `x-internal-caller: membership-worker` and the shared `@saas/contracts/billing` response shape.
2. Add `BILLING_WORKER?: Fetcher` to the membership-worker env seam, stage/prod Wrangler service bindings to same-environment billing-worker, and `billing-worker` as an Orun `dependsOn` edge for `apps/membership-worker/component.yaml`. Dev remains verify-only and should not require a live billing binding.
3. Extend `apps/billing-worker/src/router.ts` internal caller allow-list from `{ projects-worker }` to `{ projects-worker, membership-worker }`, keeping the same regex and fail-closed behavior before repository access.
4. Add a membership-owned count helper in `@saas/db/membership`, e.g. `countBillableMembers(orgId, now)` or equivalent, that counts active org members plus pending, non-expired invitations. Use this for `limit.members` comparison without loading full pages.
5. Gate `handleCreateInvitation` after role/policy authorization and before token generation / invitation insert / `invite.created` event-audit writes. Deny/error paths must write no invitation, event, or audit row and must not generate or return an invitation token.
6. Add focused tests in membership-worker, billing-worker, db-tests, and contracts if shared caller constants/types are introduced. Write the implementer report and open a GitHub PR.

1. Billing remains the only owner of billing/entitlement state. `membership-worker` must call the billing contract; it must not read `billing.*` tables or import `@saas/db/billing`.
2. Membership remains the owner of member/invitation persistence. Billing must not read `membership.*`; member-seat counting belongs in `@saas/db/membership` / `membership-worker`.
3. Preserve existing invitation authorization: actor resolution and membership policy authorization still gate invite creation. Billing entitlement checks are an additional invariant, not a replacement.
4. Do not generate invitation tokens before entitlement allow is known. Denied or service-error requests must not produce raw tokens, invitation rows, events, or audit rows.
5. Secret safety is mandatory: no plaintext credentials, API keys, bearer tokens, SQL, stack traces, provider payloads, or raw billing rows in responses/logs/reports.
6. Caller identity gating is a non-secret service-binding provenance contract. Do not add ad hoc secrets, manual shared keys, Terraform resources, or CI credential requirements.
7. Keep the PR as one coherent membership-invite entitlement-gating slice. Do not add provider adapters, billing writes, invitation idempotency, or additional product operation gates.
