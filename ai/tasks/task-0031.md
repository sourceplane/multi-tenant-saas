# Required Outcomes

1. Scaffold `apps/projects-worker`.
   - Add package, TypeScript config, ESLint config, Worker source, Wrangler
     config, and Orun component manifest following the existing
     `membership-worker` / `policy-worker` patterns.
   - The Worker must be private in stage/prod: `workers_dev: false`.
   - Stage/prod must bind:
     - `SOURCEPLANE_DB` to the verified Hyperdrive IDs;
     - `MEMBERSHIP_WORKER` to same-environment membership-worker;
     - `POLICY_WORKER` to same-environment policy-worker.
   - Add `/health` with non-secret binding/config status.
   - Add a first-class `tests/projects-worker` component/package.
   - Add Orun dependencies so first deployment ordering is safe:
     `projects-worker` should depend on `membership-worker` and `policy-worker`;
     `api-edge` should depend on `projects-worker` once it gains the
     `PROJECTS_WORKER` binding.

2. Add project public ID handling.
   - Use internal UUIDs for database and policy operations.
   - Use public API IDs at the boundary:
     - organization IDs remain `org_` + UUID hex;
     - project IDs should use `prj_` + UUID hex.
   - Reject malformed `orgId` or `projectId` as 404, not 500.
   - Do not expose raw project UUIDs in public responses, events, audit
     descriptions, or api-edge tests.

3. Implement `POST /v1/organizations/{orgId}/projects`.
   - Validate JSON body against the existing `CreateProjectRequest` shape:
     `name` required, `slug` optional.
   - Use slug rules consistent with organization creation unless a stronger
     project-specific reason appears:
     - name: 1-100 chars;
     - slug: 2-63 chars;
     - lowercase letters, numbers, hyphens;
     - starts and ends alphanumeric;
     - derive slug from name when omitted.
   - Resolve the actor from api-edge forwarded headers.
   - Call membership-worker authorization-context using the actor subject and
     raw organization UUID.
   - Call policy-worker authorize with action `project.create`,
     resource `{ kind: "project", orgId }`, and the returned memberships.
   - Fail closed: membership-context errors, policy errors, malformed envelopes,
     or policy denial must not create a project and should return safe
     `not_found` or service-unavailable style errors consistent with existing
     membership-worker behavior.
   - Persist through `@saas/db/projects`.
   - Write `project.created` event and audit entry atomically with the project
     insert using the existing transaction-capable SQL executor pattern.
   - Event/audit must use public IDs (`org_`, `prj_`) and must not include SQL,
     connection details, bearer tokens, secrets, stack traces, or raw UUIDs.
   - Return `201` with `{ project: PublicProject }` in the standard envelope.
   - Map duplicate slug/project conflicts to a safe `conflict` response.

4. Implement `GET /v1/organizations/{orgId}/projects/{projectId}`.
   - Parse both IDs from public IDs.
   - Call membership-worker authorization-context for the actor/org.
   - Call policy-worker authorize with action `project.read`,
     resource `{ kind: "project", id: projectUuid, orgId, projectId:
     projectUuid }`, and the returned memberships.
   - Fail closed and return 404 for authorization denial to avoid enumeration.
   - Load the project via `getProjectById(orgId, projectId)`.
   - Return `200` with `{ project: PublicProject }`.
   - Never query by `projectId` alone.

5. Add api-edge facade support.
   - Add `PROJECTS_WORKER` to api-edge Env and Wrangler stage/prod service
     bindings.
   - Update binding verification to assert same-environment projects-worker
     bindings.
   - Forward only:
     - `POST /v1/organizations/{orgId}/projects`
     - `GET /v1/organizations/{orgId}/projects/{projectId}`
   - Resolve identity at api-edge exactly like membership routes.
   - Forward actor subject/type/email and request/trace/idempotency headers as
     appropriate.
   - Do not forward bearer tokens to projects-worker.
   - Do not add `GET /v1/organizations/{orgId}/projects` list forwarding.

6. Keep policy and bounded-context seams clean.
   - Do not import membership repository code from projects-worker.
   - Do not query `membership.*` storage from projects-worker.
   - Do not add policy-engine actions or change policy semantics.
   - Do not add a public policy or membership facade through api-edge.
   - Projects owns project persistence; membership owns role facts; policy owns
     authorization decisions.

7. Add tests.
   - `projects-worker` tests must cover:
     - create success with authorization-context + policy allow;
     - get success with explicit `orgId + projectId`;
     - validation failures for malformed create body;
     - malformed public org/project IDs return 404;
     - missing DB, membership binding, or policy binding fails safely;

     - wrangler stage/prod bindings target same-environment projects-worker.
   - Add Orun component tests for `projects-worker-tests`.
