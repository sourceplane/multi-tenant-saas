## Objective

Add the API-key bearer actor-resolution seam through identity-worker and api-edge so organization/project/audit routes can authenticate either a user session or an active service-principal-backed API key.

This PR should make identity resolve bearer credentials into an explicit actor (`user` or `service_principal`) without adding API-key administration endpoints, UI, or role-assignment lifecycle. The slice satisfies `specs/contracts/api-guidelines.md` guidance that the public edge resolves the acting user or service principal before dispatching to internal services, while preserving bounded-context ownership: identity validates credentials; membership/policy continue to own role facts and authorization decisions.
