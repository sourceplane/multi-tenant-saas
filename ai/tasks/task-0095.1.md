# Task 0095.1 — Implementer (verifier-requested fix on PR #143)

Agent: Implementer

## Current Repo Context

Task 0095 (durable idempotency replay store at `api-edge`, B3 partial) shipped
the seam, the new `cloudflare-kv` Terraform slice, the 7-facade migration to
`replayOrExecute`, and 282/282 tests green on PR #143
(`impl/task-0095-edge-idempotency-replay-store`). Verifier ran on
2026-05-30 and returned **FAIL** — phases 1–4 PASS, phase 5 BLOCKED.

The single blocker, quoted from `ai/reports/task-0095-verifier.md`:

> `apps/api-edge/wrangler.jsonc` ships sentinel placeholder KV namespace IDs
> (`0000000000000000000000000000000a` for stage, `…000b` for prod), and
> **no pipeline step exists anywhere in this repo to substitute the real
> Terraform-output KV IDs into `wrangler.jsonc` before `wrangler deploy`
> runs.** If merged as-is, the post-merge main-CI deploy will either be
> rejected by Cloudflare ("KV namespace not found") or, worse, succeed
> with a phantom binding — every replay `kv.get`/`kv.put` throws, the
> `logKvFailure` catch-all degrades open, and the entire B3 replay store
> is silently inert in production while dashboards say everything is fine.

This is the verifier's "fixes requested by verification stay in the same PR"
case (per `agents/orchestrator.md` § PR-Sized Task Standard). Task 0095.1 is
NOT a new PR — it is an additional commit (or two) on the same branch
`impl/task-0095-edge-idempotency-replay-store` to make PR #143 mergeable.

The repo precedent for this exact problem is already in the tree:
`apps/api-edge/wrangler.jsonc` ships **real Hyperdrive IDs** for stage and
prod (`08f7c6055f544a3890a585d88fd92348` / `ab2c21c2db6245a59c91588fcac7107a`),
and `apps/api-edge/scripts/verify-bindings.mjs` asserts them at verify time.
The simplest, lowest-risk fix is to mirror that pattern for KV: apply the
`cloudflare-kv` Terraform slice once (stage + prod) to materialize the
namespaces, hardcode the resulting real 32-char hex IDs into
`wrangler.jsonc`, and extend `verify-bindings.mjs` to assert them.
