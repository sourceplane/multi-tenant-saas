# Integration Notes

- Current `apps/api-edge/wrangler.jsonc` is intentionally minimal. Use
  `wrangler.jsonc` rather than switching config formats.
- Official Cloudflare docs show Hyperdrive bindings in `wrangler.jsonc` as a
  `hyperdrive` array with `{ "binding": "...", "id": "..." }`.
- Cloudflare docs also note that bindings and `vars` are not inherited into
  named Wrangler environments; set stage/prod values explicitly.
- Existing `apps/api-edge/src/index.ts` only exposes `/health` and 404 behavior.
  Keep runtime behavior minimal. A tiny health metadata adjustment is acceptable
  only if it directly helps prove binding presence without touching the
  database.
- The existing `cloudflare-worker-turbo` deployment job currently has live
  deploy guard behavior that is not needed for this task. Do not make this PR a
  deploy fix unless non-mutating binding verification cannot otherwise be
  represented.
- The next likely task after this is runtime adapter/smoke behavior that can
  consume `env.SOURCEPLANE_DB`, followed later by an Orun-controlled Worker
  deployment task.
