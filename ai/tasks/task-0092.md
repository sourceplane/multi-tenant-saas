## Architect Brief

- Product bar: ESLint v9 baseline parity. Every `lint`-bearing
  workspace ships a runnable `eslint.config.js` so `pnpm -r lint`
  reaches every workspace and fails only on real rule violations,
  not on infrastructure gaps.
- Single shared rule source: `tooling/eslint/index.js` is the source
  of truth. Per-workspace `eslint.config.js` should be a 2-line
  re-export — the same shape used by all 17 currently-passing
  workspaces. Implementer has latitude to deviate ONLY if a workspace
  has a structural reason it cannot use the shared config (e.g. a
  framework that mandates its own flat config), and must record any
  deviation with a one-line rationale in the implementer report.
- Implementer freedom dimensions inside Constraints:
  - Add 16 `eslint.config.js` files (one per missing workspace).
  - May add per-workspace `ignores` patterns (e.g. `dist/**`,
    `coverage/**`, framework build dirs) ONLY when the workspace's
    actual file layout requires it — most won't, since
    `tooling/eslint/index.js` already ignores `dist/**`,
    `.wrangler/**`, `coverage/**`.
  - May add `@typescript-eslint/eslint-plugin` and
    `@typescript-eslint/parser` to a workspace's `devDependencies`
    ONLY if the workspace cannot resolve them transitively. Prefer the
    transitive resolution from the shared config.
- Failure modes that invalidate the PR even if `pnpm -r lint` runs:
  - Editing `tooling/eslint/index.js` itself (the rule baseline is
    already correct and is shared by 17 passing workspaces — changing
    it would surface or hide rule errors across the entire repo, way
    out of this task's scope).
  - Touching production source under `apps/**/src/**`,
    `packages/**/src/**`, `tests/**/src/**` to silence lint errors.
    Class B errors stay surfaced; they're a follow-up task.
  - Adding new runtime dependencies to any workspace.
  - Modifying any `wrangler.jsonc`, `component.yaml`, orun intent, or
    Terraform under `infra/**`.
  - Touching `infra/terraform/cloudflare-domain/**` or the
    `cloudflare ~> 4.52` provider pin (Task 0085b deferred boundary).
  - Touching the existing 17 working `eslint.config.js` /
    `eslint.config.mjs` files.
  - Regressing any jest test suite or any workspace's `typecheck`
    output.
- Recommended approach (smallest diff first):
  1. For each of the 16 missing workspaces, add an `eslint.config.js`
     that re-exports `tooling/eslint/index.js` with the canonical
     2-line shape.
  2. Run `pnpm -F <workspace> lint` per workspace to confirm:
     - Either exit 0 (workspace was clean, just missing config), OR
     - Exit 1 with rule-violation output (Class B-style — pre-existing
       errors now visible). Capture the post-scaffold lint summary in
       the report; do NOT fix the rule violations in this PR.
  3. Run `pnpm -r --no-bail lint` once at the end and record the
     before/after Class-A vs Class-B split in the report.
