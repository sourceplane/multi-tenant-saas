# Required Outcomes

1. Add project/environment contract types.
   - Add a new contracts module, expected path
     `packages/contracts/src/projects.ts`.
   - Export it from `packages/contracts/src/index.ts`.
   - Add a package subpath export such as `@saas/contracts/projects`.
   - Define public API shapes for future routes:
     - `PublicProject`
     - `CreateProjectRequest`
     - `CreateProjectResponse`
     - `GetProjectResponse`
     - `ListProjectsResponse`
     - `ArchiveProjectResponse`
     - `PublicEnvironment`
     - `CreateEnvironmentRequest`
     - `CreateEnvironmentResponse`
     - `GetEnvironmentResponse`
     - `ListEnvironmentsResponse`
     - `ArchiveEnvironmentResponse`
   - Public IDs should be modeled as strings and documented by names/types as
     public IDs. Do not add Worker ID helpers in this task.
   - Keep fields minimal and additive: `id`, `orgId`, `name`, `slug`, `status`,
     `createdAt`, `updatedAt`, and `archivedAt` where appropriate. Environment
     objects must include both `projectId` and `orgId`.

2. Add `040_projects_core` migration.
   - Create `packages/db/src/migrations/040_projects_core/up.sql`.
   - Add it to `packages/db/src/manifest.ts` with the correct SHA-256 checksum,
     sorted after `030_events_audit_core`.
   - Use bounded context `"projects"`.
   - Create a `projects` schema.
   - Create a projects table owned by this bounded context.
   - Create an environments table owned by this bounded context.
   - Every project row must carry `org_id`.
   - Every environment row must carry `org_id + project_id`.
   - Do not add foreign keys to membership tables or other bounded-context
     schemas. Cross-context foreign keys are prohibited by the repo spec.
   - Recommended statuses: `active`, `archived`.
   - Project slugs must be unique within an organization, not globally.
   - Environment slugs must be unique within `org_id + project_id`, not
     globally.
   - Add useful indexes for:
     - project list by `(org_id, created_at DESC, id DESC)`;
     - environment list by `(org_id, project_id, created_at DESC, id DESC)`;
     - project slug lookup within org;
     - environment slug lookup within org/project.
   - Keep the SQL idempotent with `CREATE SCHEMA IF NOT EXISTS`,
     `CREATE TABLE IF NOT EXISTS`, and `CREATE INDEX IF NOT EXISTS`.

3. Add a Worker-safe projects repository adapter.
   - Add `packages/db/src/projects/types.ts`.
   - Add `packages/db/src/projects/repository.ts`.
   - Add `packages/db/src/projects/index.ts`.
   - Export `@saas/db/projects` from `packages/db/package.json`.
   - Export relevant project repository types from the root `@saas/db` index
     only if that matches existing package patterns.
   - Add the explicit `@saas/db/projects` path mapping to
     `tests/db/tsconfig.json`; update Jest mapping only if needed.
   - Repository methods must require tenant scope:
     - `createProject(input)`
     - `getProjectById(orgId, projectId)`
     - `getProjectBySlug(orgId, slugLower)`
     - `listProjectsPaged(orgId, params)`
     - `archiveProject(orgId, projectId, archivedAt)`
     - `createEnvironment(input)`
     - `getEnvironmentById(orgId, projectId, environmentId)`
     - `getEnvironmentBySlug(orgId, projectId, slugLower)`
     - `listEnvironmentsPaged(orgId, projectId, params)`
     - `archiveEnvironment(orgId, projectId, environmentId, archivedAt)`
   - Do not expose any repository method that can fetch project or environment
     data by `projectId` or `environmentId` alone.
   - Follow existing repository result patterns:
     - typed success/error union;
     - safe internal errors;
     - `not_found` and `conflict` where appropriate;
     - parameterized SQL only.
   - Reuse the established cursor pagination shape from membership where
     sensible: default consumers can pass `{ limit, cursor }`; queries should
     use `(created_at DESC, id DESC)` with `limit + 1` detection.
   - Repository IDs may be raw database UUIDs internally. Public `prj_`/`env_`
     ID conversion is for the future Worker/API task, not this repository
     foundation task.

4. Preserve project isolation invariants.
   - Every SQL query for a project must include `org_id`.
   - Every SQL query for an environment must include `org_id + project_id`.
   - Tests must fail if a repository query can operate by project ID alone.
   - Environment methods must not infer organization from `project_id`.
   - Audit/event tables already support project/environment IDs; do not wire
     project events in this task.

5. Keep tests focused and strong.
   - Contract tests should prove the new public project/environment types are
     structurally usable from `@saas/contracts` and/or
     `@saas/contracts/projects`.
   - Migration tests should cover:
     - manifest includes `040_projects_core`;
     - checksum matches;
     - migration context is `"projects"`;
     - migration SQL contains `org_id` and `project_id`;
     - no foreign key references to `membership.` or other bounded-context
       schemas.
   - DB repository tests should cover:
     - `createProject` parameterized insert and conflict mapping;
     - `getProjectById` includes org ID and project ID;
     - `getProjectBySlug` scopes slug lookup by org ID;
     - `listProjectsPaged` ordering/cursor behavior;
     - `archiveProject` updates only active project rows in the route org;
     - `createEnvironment` carries org ID and project ID;
     - `getEnvironmentById` includes org ID + project ID + environment ID;
     - `listEnvironmentsPaged` includes org ID + project ID and cursor behavior;
     - `archiveEnvironment` updates only active environment rows under
       org+project;
     - safe internal error mapping.
   - Do not weaken existing migration, membership, events, identity, or runner
     tests.

6. Preserve Orun and deployment behavior.
   - No new app component.
   - No new Worker deploy.
   - No Terraform or Cloudflare changes.
   - This PR does add a database migration. PR CI should verify plan behavior.
     After merge, the verifier must confirm main CI applies the migration to
     stage and prod through the existing db-migrate Orun path.
   - Reports must not include database credentials, connection strings, Supabase
     service keys, or generated secrets.
