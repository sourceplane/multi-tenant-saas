# Required Outcomes

1. Add a destructive environment policy action.
   - Add `environment.delete` to the shared policy contract and policy engine.
   - Treat it as project-scoped and require explicit `orgId + projectId`.
   - Role semantics must mirror `project.delete`:
     - organization `owner` and `admin` may archive environments;
     - organization `builder`, `viewer`, and `billing_admin` may not;
     - matching `project_admin` may archive environments;
     - `project_builder` and `project_viewer` may not;
     - cross-org or wrong-project role facts must deny.
   - Do not use `environment.update` for archival, because archival is a
     destructive lifecycle transition. If implementation finds this action name
     conflicts with an active spec contract, write a proposal and stop before
     widening destructive access.

2. Add the projects-worker environment archive route.
   - Route:
     `DELETE /v1/organizations/{orgId}/projects/{projectId}/environments/{environmentId}`.
   - Parse `org_`, `prj_`, and `env_` public IDs and return 404 for malformed IDs.
   - Require actor headers exactly like existing project/environment routes.
   - Missing actor headers must return 401.
   - Unsupported methods on the environment item route must still return 405.

3. Enforce tenant and lifecycle invariants.
   - Confirm the parent project exists in the organization and is active before
     archiving the environment. Missing or archived parent project must return
     404.
   - Use `archiveEnvironment(orgId, projectId, environmentId, archivedAt)`.
   - Never archive or query an environment by `environmentId` alone.
   - Missing, already archived, wrong-project, or wrong-org environments must
     return 404.
   - Repository/internal failures must map to safe 503 responses.

4. Authorize through existing bounded-context seams.
   - Use membership-worker authorization-context with actor subject and raw org
     UUID.
   - Use policy-worker with action `environment.delete`.
   - Authorize with explicit environment resource:
     `{ kind: "environment", id: environmentId, orgId, projectId, environmentId }`.
   - Policy denial must return 404 to avoid resource enumeration.
   - projects-worker must not import membership repository code or query
     `membership.*` storage directly.

5. Write `environment.archived` event/audit atomically.
   - Environment archive and `environment.archived` event/audit append must
     share the transaction-capable SQL executor pattern used by project archive
     and environment create.
   - If event/audit append fails, the environment archive must not successfully
     commit.
   - Use event type `environment.archived`, source `projects-worker`, category
     `projects`, and a concise audit description.
   - Event/audit scope columns should include raw `orgId`, `projectId`, and
     `environmentId`.
   - Public-facing payloads and audit descriptions should include public
     `org_`, `prj_`, and `env_` IDs and must not expose raw UUIDs, database
     connection details, bearer tokens, API keys, SQL, stack traces, or other
     secret material.

6. Update api-edge project facade.
   - Forward `DELETE` on
     `/v1/organizations/{orgId}/projects/{projectId}/environments/{environmentId}`
     to `PROJECTS_WORKER` after identity resolution.
   - Preserve existing `GET` behavior for environment item routes.
   - Forward actor headers, request ID, traceparent, and idempotency-key as
     appropriate.
   - Continue not forwarding bearer tokens.
   - Do not forward a body for `DELETE`.
   - Missing `PROJECTS_WORKER` must return a safe 503.

7. Add focused tests.
   - contracts/policy tests must cover `environment.delete` as a known action.
   - policy-engine tests must cover `environment.delete` role matrix, project
     scope requirement, cross-org denial, wrong-project denial, and project role
     behavior.
   - projects-worker tests must cover successful archive, `environment.delete`
     action/resource shape, malformed IDs, missing actor, missing bindings,
     missing/archived parent project, missing/already archived/wrong-project
     environment, policy denial, membership-context failure, event/audit append
     failure rollback behavior, safe 503 mappings, and response/event redaction.
   - api-edge tests must cover DELETE forwarding, no bearer forwarding, no body
     forwarding, actor/request/trace/idempotency header forwarding, missing
     `PROJECTS_WORKER`, method restrictions, and existing environment GET route
     preservation.
