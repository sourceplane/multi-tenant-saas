Last updated: 2026-05-19

## Active Decisions

- Use `specs/**` as the authoritative spec pack for current work. Do not read or apply `specs-v2/**` unless the task is product-specific Git catalog or CI intelligence work.
- Treat the repo-relative `ai/` directory as the active planning state location for this checkout. The orchestrator text uses `/ai/...` paths conceptually, while `agents/agent-loop.sh` reads `ai/state.json` and `ai/waiting_for_input.md` relative to the workspace.
- Use `specs/orun-golden-path.md` as the short shared Orun context for all implementer and verifier agents.
- Treat `../aws-admin` as the reference repo for Terraform-oriented Orun behavior:
  - Orun runtime version `v2.1.0`;
  - `sourceplane/orun-action@v1.2.0`;
  - `dev`, `stage`, `prod` environments;
  - `parameterDefaults.terraform`;
  - Terraform components with `spec.parameters`, `plan-only` default profile, `apply` profile rules, colocated README docs, and S3 backend shape.
- AWS IAM roles and shared Terraform state buckets are owned by `aws-admin`. `multi-tenant-saas` consumes the repo-scoped role created there.
- Terraform state for `multi-tenant-saas` must use the shared S3 buckets `sourceplane-<env>`, not Cloudflare R2.
- Supabase database credentials and connection details must be generated by Terraform and stored in AWS Secrets Manager under `<org>/<repo>/<component>/<env>`.
- Cloudflare/Supabase human-provided resources discovered in Task 0002 remain useful historical context, but new infrastructure work follows the AWS/S3/Secrets Manager path unless a later spec explicitly says otherwise.
