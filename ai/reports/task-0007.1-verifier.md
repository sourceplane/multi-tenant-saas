## Risk Notes

### Missing `dependsOn` edge from `db` to `db-tests`

**Decision: Accepted limitation, not a merge blocker.**

`specs/repo.md` states: "Deployable, package, and infra components must declare `dependsOn` edges to the test components that gate them."

Evidence for acceptance:
- `packages/contracts/component.yaml` (the reference shared package) also has no `dependsOn` on its test component — this is the established repo pattern.
- `packages/db` subscribes to `dev`, `stage`, `prod`. `tests/db` subscribes only to `dev`. A `dependsOn` edge from `db` to `db-tests` would be unsound for `stage` and `prod` instances where `db-tests` has no job. Orun does not currently support environment-scoped dependency edges.
- The verifier has confirmed that `db-tests.dev.verify` passed independently in CI run `26220171323`.
- The implementer report documents this gap explicitly with reasoning.

A spec proposal should be filed to add environment-scoped `dependsOn` support in Orun or to clarify the spec for environment-subscription-mismatched package/test pairs. This is deferred, not blocking.
